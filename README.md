# MMAP-Net: Missing-Aware Multimodal Fusion for Multi-Label Retinal Disease Classification with Score-Calibrated Deferral

Official reproducibility repository for the article:

> **MMAP-Net: Missing-Aware Multimodal Fusion for Multi-Label Retinal Disease Classification with Score-Calibrated Deferral**
> Accepted for publication in *Expert Systems with Applications* on July 10, 2026.

This repository contains the released prediction exports, patient-level split summary, calibration and
deferral artifacts, manuscript result tables, and scripts used to verify the reported results and
regenerate the quantitative figures that do not require raw fundus images or trained model checkpoints.

The repository follows the terminology and experimental definitions used in the accepted article:

* **score-calibrated deferral** terminology throughout;
* **single external-device shift check** terminology throughout;
* **controlled MCAR and label-coupled missingness analyses**, without claiming verified MNAR robustness;
* **canonical MMAP-Net = Heads A/B only**; the auxiliary hypertension branch is reported only as an exploratory negative result.

## Quick start

```bash
pip install -r requirements.txt
python scripts/verify_consistency.py
python scripts/generate_figures.py
```

`verify_consistency.py` checks the released per-sample prediction exports against the corresponding
article tables for the experiments for which per-sample outputs are redistributed.

`generate_figures.py` regenerates the quantitative result figures from the released CSV and prediction
files.

The workflow diagram, architecture diagram, and Grad-CAM++ panels are included in `figures/`, but they
are not regenerated by `generate_figures.py`. The workflow and architecture figures are manuscript
diagrams, while Grad-CAM++ generation requires access to the original images and trained checkpoint.

Detailed reproduction instructions are provided in [`REPRODUCE.md`](REPRODUCE.md), and descriptions of
the released data files are provided in [`DATA.md`](DATA.md).

## Evaluation protocols

### Protocol 1: canonical performance and operating points

Protocol 1 uses the validation-selected seed-42 checkpoint, `T=20` MC Dropout, and the held-out BRSET
test split (`n=3,266`).

This protocol is used for:

* headline classification results;
* validation-fitted calibration;
* score-calibrated deferral;
* subgroup analysis;
* validation-locked diabetic retinopathy operating points.

The canonical macro-AUROC under Protocol 1 is **0.9673**.

### Protocol 2: matched component comparisons

Protocol 2 uses a single deterministic forward pass (`T=1`) under matched experimental conditions.

This protocol is used for:

* model ablations;
* component comparisons;
* matched-retraining analyses;
* three-seed reproducibility assessment.

The full-model macro-AUROC under Protocol 2 is **0.9289**.

Absolute metric values should not be compared directly across the two protocols because `T=20`
Monte Carlo averaging is used only in Protocol 1.

## Calibration and deferral scope

The accepted article reports a deployment-oriented calibration protocol based on label-specific
isotonic regression.

The calibrators are:

1. fitted using the validation split;
2. locked without access to test labels;
3. applied unchanged to the held-out BRSET test split and the external mBRSET dataset.

The reported validation-fitted macro-ECE on the held-out BRSET test split is **0.019**.

The deferral threshold is also selected using the validation split and subsequently applied unchanged
to the held-out BRSET test split and mBRSET.

The proposed mechanism is a **score-calibrated deferral rule**. It is not a formal conformal-prediction
procedure and does not claim finite-sample coverage guarantees under deployment or distribution shift.

## Released prediction exports

* `data/predictions_test_seed42.csv`
  Held-out BRSET test probabilities and targets for the four primary retinal labels.

* `data/validation_probabilities_seed42.csv`
  Validation-split probabilities used to fit the label-specific isotonic calibrators and select the
  score-calibrated deferral threshold.

* `data/predictions_cache_verified.pkl`
  Per-sample prediction cache for `mmap_full` and `mmap_no_missingness`, used for reproducible
  Protocol 2 consistency checks.

* `data/patient_split_map.csv`
  Patient-level train, validation, and test assignments using hashed patient identifiers.

Per-sample caches for the remaining ablation variants are not redistributed. Their aggregate results
are reported in `data/table06_ablation.csv`.

## Dataset and privacy note

This repository does not redistribute patient images, clinical records, or directly identifying
information.

BRSET and mBRSET remain available through their original source repositories and are subject to their
respective access, licensing, and credentialing conditions.

The files released here contain only:

* anonymized model prediction outputs;
* aggregate result tables;
* calibration and deferral artifacts;
* hashed patient-level split identifiers;
* scripts for consistency verification and quantitative figure regeneration.

The hashed identifiers are included only to document patient-level partitioning and cannot be used to
recover the original patient identifiers from this repository.

---

## Reproducibility scope

The repository supports independent verification of the released numerical outputs, including:

* consistency between redistributed prediction exports and reported metrics;
* regeneration of quantitative figures from released result files;
* inspection of patient-level partition assignments;
* inspection of validation-fitted calibration and deferral artifacts;
* comparison of canonical and ablation protocols;
* inspection of controlled missingness and external-device analyses.

The repository does not provide a fully executable end-to-end retraining pipeline because the original
fundus images and trained checkpoints are not redistributed.

Accordingly, this repository should be interpreted as a **result-verification and quantitative
reproducibility package**, rather than as a redistribution of the complete training environment.
