# MMAP-Net: Missing-Aware Multimodal Fusion for Multi-Label Retinal Disease Classification with Score-Calibrated Deferral

Companion repository for the revised manuscript submitted to *Expert Systems with Applications*.
It contains the released prediction exports, patient-level split summary, calibration-threshold
artifacts, manuscript result tables, and scripts used to regenerate the quantitative result figures
that do not require raw fundus images or trained checkpoints.

The repository has been updated to match the revised manuscript terminology:

- **score-calibrated deferral** terminology throughout;
- **single external-device shift check** terminology throughout;
- **controlled MCAR and label-coupled missingness analyses**, not verified MNAR robustness;
- **canonical MMAP-Net = Heads A/B only**; the auxiliary hypertension branch is reported only as an exploratory negative result.

## Quick start

```bash
pip install -r requirements.txt
python scripts/verify_consistency.py
python scripts/generate_figures.py
```

`verify_consistency.py` checks the released per-sample prediction exports against the
corresponding manuscript tables where per-sample outputs are redistributed. `generate_figures.py`
regenerates the quantitative result figures from the released CSV/prediction files.

The workflow diagram, architecture diagram, and Grad-CAM panels are included in `figures/` but are
not regenerated by `generate_figures.py`. The first two are manuscript vector diagrams; Grad-CAM
requires the raw images and trained checkpoint.

## Evaluation protocols

- **Protocol 1 (canonical performance and operating points):** validation-selected seed-42
  checkpoint, `T=20` MC Dropout, held-out BRSET test split (`n=3,266`). This protocol is used for
  headline classification results, validation-fitted calibration, deferral behavior, subgroup analysis,
  and validation-locked DR operating points. Canonical macro-AUROC = **0.9673**.

- **Protocol 2 (matched component comparisons):** single deterministic forward pass (`T=1`) under
  matched conditions. This protocol is used for ablation and component comparisons. Full-model
  macro-AUROC = **0.9289**.

Absolute values should not be compared across protocols because `T=20` MC averaging is used only in
Protocol 1.

---

## Calibration and deferral scope

The revised manuscript reports one deployment-relevant calibration protocol: label-specific isotonic
regression fitted on the validation split and applied unchanged to the held-out test split. The reported
macro-ECE is **0.019**. The deferral threshold is also fitted on the validation split and then applied
unchanged to the held-out BRSET test split and to mBRSET.

The deferral mechanism is a **score-calibrated deferral rule**. It is not formal conformal prediction and
it does not claim finite-sample deployment coverage guarantees.

## Released prediction exports

- `data/predictions_test_seed42.csv`: held-out BRSET test probabilities and targets for the four primary labels.
- `data/validation_probabilities_seed42.csv`: validation-split probabilities used to fit isotonic calibrators and deferral thresholds.
- `data/predictions_cache_verified.pkl`: per-sample prediction cache for `mmap_full` and `mmap_no_missingness`, used for reproducible Protocol 2 checks.
- `data/patient_split_map.csv`: patient-level partition assignment with hashed patient identifiers.

Per-sample caches for the remaining ablation variants are not redistributed; their aggregate metrics are
provided in `data/table06_ablation.csv`.

---

## Dataset note

This repository does not redistribute patient images. BRSET and mBRSET are publicly available from their
source repositories under their own access and credentialing requirements. The released CSV files contain
anonymized prediction outputs, aggregate result tables, and hashed split identifiers only.
